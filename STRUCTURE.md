# NEPSE-Analysis Monorepo Structure

This document outlines the standardized, physical project directory structure for the NEPSE-Analysis platform.

## 📂 Physical Layout

### 🚀 `apps/` (User-Facing Applications)
- **`api/`** (formerly `backend/`): Node.js/Express/Prisma backend server.
    - `src/controllers/`: Business logic handlers.
    - `src/services/`: Core domain logic (Analytics, Auth).
    - `prisma/`: Database schema and migrations.
- **`mobile/`** (formerly `nepse_app/`): Flutter/Dart mobile application.
    - `lib/models/`: Shared data structures.
    - `lib/services/`: API interaction layer.
    - `lib/widgets/`: Atomic UI components.

### 📦 `packages/` (Shared Modules)
- **`client_sdk/`** (formerly `dart_client/`): Core internal library for interacting with the NEPSE API.
    - `lib/`: Shared Dart models and client logic.
    - `web/`: High-fidelity web dashboard.

### 🛠️ `tools/` (Infrastructure & Automation)
- **`analysis_engine/`**: Python-based technical analysis and AI marketing systems.
- **`scripts/`**: Automation and execution batch files.

### 📜 `docs/`
- Documentation, architecture diagrams, and structure guides.

---

## 🏗️ Standards

### Naming Conventions
- **Directories**: `kebab-case` (e.g., `client-sdk`).
- **Files**: `snake_case` (e.g., `auth_controller.js`).
- **Widgets**: `PascalCase` (e.g., `GlassContainer.dart`).

### Dependency Rules
- `apps/mobile` depends on `packages/client_sdk` via relative pathing.
- Root management is handled via **npm workspaces** defined in `./package.json`.
